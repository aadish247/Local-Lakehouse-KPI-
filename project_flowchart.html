Local Lakehouse KPI Project Flowchart

Introduction

This flowchart explains the Local Lakehouse KPI project step-by-step, including definitions and detailed explanations of each component.

Raw Data

Data is ingested into DuckDB, a lightweight database for local storage and querying.

↓

DuckDB

A fast, in-process SQL OLAP database system used for storing and querying data locally.

↓

dbt (Data Build Tool)

Transforms raw data into structured tables using SQL-based models.

↓

Transformed Tables

Fact and dimension tables (e.g., fact_sales, dim_customer, dim_product) are created.

↓

Exported Files

Tables are exported to Parquet and CSV formats for analysis.

↓

Power BI

Visualizes the exported data by creating interactive dashboards and charts.

↓

Python Charts

Generates charts as JPG files for documentation and reporting.

Definitions

  • Lakehouse: A unified platform that combines the features of data lakes and data warehouses.
  • DuckDB: A fast, in-process SQL OLAP database system.
  • dbt: A tool for data transformation and modeling using SQL.
  • Power BI: A business analytics tool for creating visualizations and dashboards.
  • Fact Table: A table that contains measurable, quantitative data (e.g., sales).
  • Dimension Table: A table that contains descriptive attributes related to fact data (e.g., customers, products).

Pipeline Overview

The pipeline consists of the following steps:

  1. Data Ingestion: Raw data is loaded into DuckDB.
  2. Data Modeling: dbt transforms the raw data into structured tables.
  3. Data Export: Tables are exported to Parquet and CSV formats for analysis.
  4. Visualization: Power BI creates dashboards from the exported data.
  5. Documentation: Python generates charts for reporting purposes.

Conclusion
